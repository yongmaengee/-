## 2. 가장 중요한 발견 — 데이터 누수(Label Leakage) 위험

### 2-1. 위험등급은 누적 날씨 피처로부터 룰 기반 산출
- `magnitude_score`, `날씨_위험도_복합` ↔ `주의이상` 상관 r=0.557
- `risk_heat_process` ↔ `주의이상` r=0.555
- 한파_연속_7d ≥ 4일 케이스 → **100% 주의이상 분류**
- 위험등급 Q5(상위 20%) 케이스 → 주의이상률 28~33%
- → **위험등급 예측 모델에 magnitude_score / 날씨_위험도_복합 / risk_* 류 피처를 함께 넣으면 사실상 라벨을 입력으로 쓰는 것**

### 2-2. 사고유형(추락/붕괴/...) 플래그는 공종×날씨 결정론적 룰
- 가설/철골/수장공사 → 추락_낙하 = 100%
- 철근콘크리트공사 → 구조물_균열 = 100%
- 해체철거/토공사/건축토공사 → 붕괴_도괴 = 100%
- 전기설비/도장공사 → 화재_폭발 = 100%
- 강우/강설 날씨 → 침수_수해, 붕괴_도괴 = 100%
- 강풍 날씨 → 추락_낙하 = 100%
- → **사고유형 플래그는 입력으로도 출력으로도 쓰면 안 됨** (공종+날씨에서 자동 도출되므로)

### 2-3. 중대재해는 누적 날씨와 거의 무상관
| 피처 | 중대재해 점이상관 r | 주의이상 점이상관 r |
|---|---|---|
| weather_scale_interact | 0.083 | 0.383 |
| risk_wind_scaffold | 0.017 | 0.340 |
| 폭염_연속_7d | -0.0003 | 0.462 |
| 누적강수_7d | -0.007 | 0.086 |
| magnitude_score | 0.0005 | 0.557 |

→ 누적 날씨로 "위험등급"은 잘 예측되지만(이미 룰로 정의됨), **실제 사망사고와는 약한 신호**.

## 5. 모델 설계 권고

### 5-1. 두 개의 분리된 타깃 정의
사용 사례에 따라 분리해서 학습 권장:
- **타깃 A — 위험등급(주의이상)** : 룰 기반 라벨이지만 사용자 인터페이스에서 친숙. 룰을 그대로 모방하는 lightweight 모델로 충분 (트리 깊이 얕은 것).
- **타깃 B — 중대재해(사망≥1)** : 실질적으로 막아야 할 사건. 신호가 약하고 불균형 4.1%. 메인 타깃으로 채택 권장.
- **타깃 C — C_RPI 회귀** : magnitude_score/날씨_위험도_복합과 r=0.84로 사실상 동일. 별도 라벨로 보지 말고 입력 피처와 같이 다루어 단조 제약(monotonic) 모델에서 사용.

### 5-2. 입력 피처 화이트/블랙리스트
**제외 (라벨 누수)**: `위험등급`, `C_RPI`, `magnitude_score`, `날씨_위험도_복합`, `risk_heat_process`, `risk_rain_progress`, `risk_wind_scaffold`, `risk_cold_concrete`, `weather_scale_interact`, `heat_*_exposure`, 사고유형 7종 플래그, 사상자 컬럼 전체.

**제외 (상수)**: `강풍_연속_7d`.

**입력 후보**:
- 누적: `누적강수_7d`, `누적풍속_7d`, `기온변동성_7d`, `폭염_연속_7d`, `한파_연속_7d`, `강수_연속_7d`, `ATD_7d`, `TSI_3d`
- 당일: `최고기온`, `최저기온`, `일강수량`, `평균 풍속`, `날씨` 카테고리
- 컨텍스트: `시도구분`, `공사대분류`, `공종(소분류)`, `공정율_수치`, `공사비_순위`, `소규모현장`, `고위험공종`
- 시간: `월`, `시`, `요일`, (계절은 월에서 도출)

### 5-3. 추천 모델 구조
누적 날씨로부터 "이슈 리스트"를 출력하는 모델은 단일 분류기가 아니라 **2단 구조** 권장:

1. **위험 점수기 (Scorer)** — Gradient Boosting (LightGBM/XGBoost)으로 `중대재해` 확률 추정.
   - 불균형 대응: `scale_pos_weight ≈ 23` 또는 focal loss
   - SHAP으로 피처 기여도 산출 → 이슈 사유 설명에 활용

2. **이슈 룰 추출기 (Rule miner)** — 학습된 모델 위에 RuleFit 또는 결정트리(depth=3~4)로 사람이 읽을 수 있는 룰 N개 추출.
   - 위 §4의 결합 룰들이 이미 강한 후보. 추가로 데이터에서 자동 학습.
   - 출력 예: `[공종=철골공사 AND 강수_연속_7d≥3] → 위험도 10.98%, 권고: 안전대 점검·고소작업 지연 검토`

3. **이슈 리스트 후처리** — 사용자가 입력한 누적 날씨 + 현장 컨텍스트에 활성화된 룰을 Lift 내림차순으로 N개 노출.

### 5-4. 검증 설계
- **시계열 분할 필수**: 2019-07~2024-12 학습 / 2025-01~2025-12 검증.
- 무작위 K-fold는 동일 현장·인근 일자가 학습/검증에 같이 들어가 overfitting을 가린다.
- 평가 지표: PR-AUC (불균형이라 ROC-AUC는 낙관적), Recall@TopK (상위 K% 알람 시 잡힌 중대재해 비율), Lift@K.

### 5-5. 한계 및 추가 자료 필요성
- **사고가 없는 일자/현장 데이터가 없음** — 이 데이터셋은 모두 "사고가 발생한" 케이스. 따라서 학습 가능한 것은 "사고가 났을 때 그것이 중대재해인가?"이지 "이런 날씨에 사고가 더 많이 나는가?"는 답할 수 없다.
- 후자(사고 빈도 예측)를 하려면 **현장×일자 단위의 무사고 행 + 작업 진행여부**가 필요.
- 외부 데이터 결합 후보: KMA 시간단위 기상실측, 행정구역 단위 공사현장 수, 작업일지(작업 중단 여부).
