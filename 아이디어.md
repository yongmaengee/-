# 아이디어

#### TL;DR

> 시설안전에 관한 리스크를 기후변화에 따라서 상관성을 분석하여,
현재시점의 현장에서 실시간 기후에 대해 발생할만한 이슈 및 리스크들을 정리해 알려주는 시스템
> 

### Overview:

```smalltalk
오프라인 (학습)
  기후 피처 × 사고원인 → Feature-Issue 가중치 행렬

온라인 (추론)
  실시간 기후 피처 → 이탈량 계산 → 누적 → 가중치 행렬과 곱 → 이슈 체크리스트
```

기후 피처 이탈 → 누적 exceedance 벡터 → Feature-Issue Attention → 이슈별 magnitude + 근거 피처 → 체크리스트

핵심: **오프라인 상관관계 학습**과 **온라인 이상 누적 모니터링**

---

## 구체화

input feature : 시계열 날씨 데이터(2000~2020)

label : 시설안전과 관련한 위험요소 및 사고사례

architecture: Attention기반 sequence모델(tiny transformer) 또는 사전학습 트랜스포머

### Label 설계

-3가지 층위로 분리

#### layer 1 - issue Category

- 사용 데이터: **사고원인별 분류(국토안전 빅데이터)**

```smalltalk
지반침하 / 사면붕괴 / 구조물 균열 / 침수피해 / 기타
→ multi-label: 하나의 기후 패턴이 여러 이슈를 동시에 유발할 수 있음
```

#### Layer 2 - Magnitude(누적 이탈량(기후변화 누적))

학습데이터에서 사고 기록과 얼마나 겹치는지 상관관계 구하기

```python
# 개념적 코드
exceedance = max(0, feature_value - threshold)
cumulative_exceedance = integrate(exceedance, time_window)
```

*개별 사고 “발생 여부”가 Label이 아닌, 피쳐가 역치를 얼마나 오래 넘었는지에 대한 지표

#### Layer 3 - Feature Attribution (체크리스트 근거)

어떤 피쳐가 이 이슈를 유발하고 있는지 알려주는 Attribution.

SHAP/Attention weight로 추출

---

Attention 설계:

```smalltalk
Query = 현재 시점의 [피처 이탈 벡터] (shape: n_features)
Key   = 역사적 사고 패턴 임베딩   (shape: n_issues × n_features)
Value = 이슈별 체크리스트 항목 가중치

Attention(Q, K, V) → 각 이슈에 대한 활성화 강도 + 어떤 피처가 기여했는지
```
